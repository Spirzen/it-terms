## LAMP

LAMP — стек программного обеспечения с открытым исходным кодом для веб-разработки, состоящий из:
- **L**inux (операционная система),
- **A**pache (веб-сервер),
- **M**ySQL (реляционная СУБД),
- **P**HP/**P**erl/**P**ython (языки серверной разработки).  
Альтернативные варианты: LEMP (Nginx вместо Apache), WAMP (Windows вместо Linux).

## LAN

LAN (Local Area Network) — локальная вычислительная сеть, объединяющая устройства в пределах ограниченной географической зоны (офис, дом, здание). Характеризуется высокой скоростью передачи данных, низкой задержкой и использованием протоколов Ethernet/Wi-Fi. В играх — режим мультиплеера без интернета.

## Language Integrated Query (LINQ)

Language Integrated Query (LINQ) — встроенная в C# и другие языки .NET модель запросов к данным: коллекции в памяти (`IEnumerable`), реляционные БД через `IQueryable` (EF Core, LINQ to DB), XML (`System.Xml.Linq`), а также сторонние провайдеры. Два эквивалентных стиля — синтаксис запросов (`from` / `where` / `select`) и цепочки методов (`Where`, `Select`, `GroupBy`). Учебная статья — [LINQ в C#](/encyclopedia/5-languages/5-05-csharp/29), полный API — [Справочник по LINQ](/encyclopedia/5-languages/5-05-csharp/291).

## LBA

LBA (Logical Block Addressing — логическая адресация блоков) — метод обращения к данным на блочных устройствах (жёстких дисках, SSD), при котором каждый сектор (блок) нумеруется последовательно одним целым числом, начиная с 0. Пришла на смену CHS (Cylinder-Head-Sector — цилиндр-головка-сектор), которая была сложна для больших дисков и зависела от физической геометрии. LBA абстрагирует программное обеспечение от аппаратных деталей: ОС и приложения работают с "плоским" адресным пространством, а контроллер диска сам преобразует LBA в физические координаты. 

---

## LBA-адрес

LBA-адрес — конкретное числовое значение (например, 1024), указывающее позицию логического блока на устройстве хранения. В контексте ОС и приложений это — адрес данных, не зависящий от физического расположения. Файловые системы отображают логические адреса файлов (например, смещение в байтах) на LBA-адреса секторов через метаданные (таблицы размещения, extent-карты). Утилиты низкого уровня (dd, hdparm, nvme-cli) позволяют читать/писать напрямую по LBA-адресам, что полезно для диагностики, восстановления данных или криминалистики. Для SSD LBA — виртуальный адрес: контроллер преобразует его в физический адрес NAND-ячеек с учётом wear leveling и garbage collection.

## LC

LC — Least Connections, алгоритм балансировки нагрузки. При этом методе новый запрос направляется на сервер с наименьшим числом активных соединений на момент принятия решения.

## LDAP

LDAP реализует доступ к иерархическим справочникам через сетевой протокол. Справочник хранит информацию об объектах — пользователях, группах, устройствах — в древовидной структуре с атрибутами. Каждый объект имеет уникальный отличительный путь, составленный из атрибутов в обратном порядке. Запросы выполняются через операции поиска, добавления, изменения и удаления. Фильтры поиска используют префиксную нотацию для сравнения атрибутов. Каталог оптимизирован для частого чтения и редкой записи. Репликация поддерживает распределённые копии справочника для отказоустойчивости.

## Library

Library (библиотека) — совокупность предварительно скомпилированного кода (функций, классов, ресурсов), предназначенная для повторного использования в программных проектах. ## LuGre

LuGre (Lund-Grenoble) — математическая модель трения "сухого скольжения" (dry friction), разработанная в 1990-х годах для точного описания эффектов:
	- Статическое трение (stiction — сила, необходимая для сдвига неподвижного объекта),
	- Предельное трение (breakaway — пик перед началом движения),
	- Падающая характеристика (frictional lag — уменьшение силы трения при малых скоростях),
	- Гистерезис (разная сила при ускорении и замедлении).

Модель основана на представлении контакта как упругой "щётки" из микроволокон, деформирующихся при движении. LuGre широко используется в мехатронике и робототехнике для компенсации трения в системах управления (например, в станках с ЧПУ, манипуляторах), так как простые модели (Кулон + вязкое трение) недостаточно точны на малых скоростях. Параметры модели идентифицируются экспериментально.

LXC (Linux Containers) — технология операционной виртуализации в Linux, позволяющая запускать изолированные пользовательские пространства (контейнеры) с общим ядром хост-системы. Предшественник и основа для Docker и systemd-nspawn.

## LXD

LXD — демон и CLI-интерфейс для управления LXC-контейнерами, разработанный Canonical. Предоставляет REST API, поддержку образов, миграции и сетевых профилей, упрощая использование контейнеров как виртуальных машин.
