## WAR

Write-After-Read hazard (опасность "запись после чтения"): тип зависимости данных в конвейерных и параллельных вычислениях, возникающий, когда инструкция пытается записать значение в регистр или ячейку памяти до того, как предыдущая инструкция завершит чтение из этого же места. Может привести к некорректному результату без применения техник переименования регистров или барьеров.

Включает две основные сеттинги: фэнтезийный Warhammer Fantasy (ныне заменён Age of Sigmar) и научно-фантастический Warhammer 40,000. Отличается мрачной, сатирической эстетикой и детализированным лором.

## Web

World Wide Web (всемирная паутина): информационная система поверх интернета, основанная на гипертекстовых документах, связанных между собой гиперссылками и доступных по протоколу HTTP/HTTPS. Изобретена Тимом Бернерс-Ли в 1989 году. Не следует путать с интернетом — Web является одним из его сервисов (наряду с email, FTP и др.).

---

## Web API

Web API: программный интерфейс, доступный через веб-протоколы (обычно HTTP/HTTPS), позволяющий клиентам (браузерам, мобильным приложениям, другим сервисам) взаимодействовать с серверной логикой. Может быть реализован в стиле REST, GraphQL, gRPC или SOAP. Обеспечивает интеграцию, композицию сервисов и разработку клиент-серверных приложений.

---

## WebAssembly (Wasm)

WebAssembly: бинарный формат инструкций, разработанный W3C для безопасного, портативного и эффективного выполнения кода в веб-браузерах. Позволяет запускать приложения, написанные на языках низкого уровня (C, C++, Rust), со скоростью, близкой к нативной. Может использоваться как универсальный runtime (например, в edge-вычислениях).

---

## Webhook

Webhook: механизм пользовательского HTTP-обратного вызова, позволяющий одной системе уведомлять другую о событии в реальном времени. В отличие от polling, где клиент периодически опрашивает сервер, webhook отправляет данные сразу по наступлению события. Используется в интеграциях (GitHub, Slack, Stripe) для автоматизации рабочих процессов.

---

## WebSocket

WebSocket: протокол связи поверх TCP, обеспечивающий полнодуплексный, постоянный канал между клиентом и сервером через одно TCP-соединение. В отличие от HTTP, который требует нового запроса для каждого обмена, WebSocket позволяет отправлять сообщения в любую сторону без накладных расходов на заголовки. Используется в чатах, стриминге, онлайн-играх и real-time dashboards.

---

## WebGL

WebGL (Web-based Graphics Library) — кроссплатформенный API для отображения 3D-графики в браузере, разрабатываемый некоммерческой организацией Khronos Group. WebGL использует язык программирования шейдеров GLSL. WebGL исполняется как элемент HTML5 и поэтому является полноценной частью объектной модели документа (DOM API) браузера. Может использоваться с любыми языками программирования, которые умеют работать с DOM API, например, JavaScript, Rust, Java, Kotlin и другими. Все ведущие разработчики браузеров Google (Chrome), Mozilla (Firefox), и Apple (Safari) являются членами Khronos и реализуют WebGL в своих браузерах. За счёт использования низкоуровневых средств поддержки OpenGL часть кода на WebGL может выполняться непосредственно на видеокартах. WebGL — это контекст элемента canvas HTML, который обеспечивает API 3D-графики без использования плагинов. Первая спецификация была выпущена 3 марта 2011 года. Современная версия 2.0 (несовместима с версией 1.0) доступна с 27 февраля 2017 года.

## WHOIS

WHOIS: протокол и служба запроса информации о доменных именах, IP-адресах и автономных системах (AS). Возвращает данные о регистранте, администраторе, технических контактах, датах регистрации и DNS-серверах. Стандартизирован IETF (RFC 3912). В последние годы многие данные скрыты из-за требований GDPR.

## Wi-Fi

Wi-Fi — технология беспроводной локальной сети с устройствами на основе стандартов IEEE 802.11. Логотип Wi-Fi является торговой маркой Wi-Fi Alliance. Термин Wi-Fi считается сокращением слов Wireless Fidelity, что дословно можно перевести как "беспроводная правильность" или "беспроводная точность". Но чаще термин никак не расшифровывается. При этом один из основателей "Wi-Fi Alliance" заявил, что буквосочетание Wi-Fi было придумано как маркетинговое название новой технологии, оно изначально не было аббревиатурой и ничего не означало.

## Windows

Windows: семейство проприетарных операционных систем от Microsoft, предназначенных для настольных ПК, серверов, мобильных устройств и встраиваемых систем. Наиболее распространённые версии: Windows 10, Windows 11 (настольные), Windows Server 2022. Ядро основано на гибридной архитектуре (микроядро + монолитные компоненты). Доминирует на рынке персональных компьютеров.

## Windows API

Windows API (англ. application programming interfaces) — общее наименование набора базовых функций интерфейсов программирования приложений операционных систем семейств Microsoft Windows корпорации Майкрософт. Предоставляет прямой способ взаимодействия приложений пользователя с операционной системой Windows. Для создания программ, использующих Windows API, корпорация Майкрософт выпускает комплект разработчика программного обеспечения, который называется Platform SDK и содержит документацию, набор библиотек, утилит и других инструментальных средств для разработки.

## Windows Forms

Windows Forms — интерфейс программирования приложений (API), отвечающий за графический интерфейс пользователя и являющийся частью Microsoft .NET Framework. Данный интерфейс упрощает доступ к элементам интерфейса Microsoft Windows за счет создания обёртки для существующего Win32 API в управляемом коде. Причём управляемый код — классы, реализующие API для Windows Forms, не зависят от языка разработки. То есть программист одинаково может использовать Windows Forms как при написании ПО на C#, C++, так и на VB.Net, J# и др.

## Windows Presentation Foundation (WPF)

Windows Presentation Foundation (WPF) — аналог WinForms, система для построения клиентских приложений Windows с визуально привлекательными возможностями взаимодействия с пользователем, графическая (презентационная) подсистема в составе .NET Framework (начиная с версии 3.0), использующая язык разметки XAML.

## Windows Subsystem for Linux (WSL)

Windows Subsystem for Linux: совместимый подсистемный слой в Windows 10 и 11, позволяющий запускать двоичные файлы Linux (ELF64) непосредственно в Windows без виртуальной машины. WSL 2 использует лёгкий виртуальный Hyper-V для запуска полноценного ядра Linux. Поддерживает интеграцию с файловой системой Windows, графические приложения (WSLg) и разработку в Unix-среде.

## Wi-Fi

Wi-Fi: сертификационный бренд и набор стандартов беспроводной локальной сети на основе IEEE 802.11. Обеспечивает высокоскоростную связь в пределах ограниченной зоны (дом, офис). Работает в диапазонах 2.4 ГГц, 5 ГГц и 6 ГГц (Wi-Fi 6E/7). Управление сертификацией осуществляет Wi-Fi Alliance.
