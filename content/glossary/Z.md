## Z-order

Z-порядок (Z-ordering): метод упорядочивания графических объектов по глубине в двумерной графике, определяющий, какие элементы отображаются поверх других. Объекты с бо́льшим Z-значением рисуются позже и, следовательно, перекрывают объекты с меньшим значением. Используется в UI-фреймворках, игровых движках и компоновке окон.

---

## Z-buffer

Z-буфер (буфер глубины): буфер кадра, используемый в 3D-рендеринге для хранения глубины каждого пикселя относительно камеры. Позволяет корректно отображать только ближайшие поверхности, устраняя проблемы наложения при рендеринге непрозрачных объектов. Работает в связке с цветовым буфером (framebuffer) и является стандартной частью конвейера рендеринга в OpenGL, DirectX и Vulkan.

## ZK-SNARKs

Zero-Knowledge Succinct Non-Interactive Argument of Knowledge: криптографический метод доказательства, при котором одна сторона (доказатель) может убедить другую (проверяющего) в истинности утверждения, не раскрывая самой информации, на которой это утверждение основано. Обладает свойствами: краткости (succinct), отсутствия интерактивности (non-interactive) и нулевого разглашения (zero-knowledge). Применяется в приватных блокчейнах (Zcash), верифицируемых вычислениях и защите данных.

## Zabbix

Zabbix: система мониторинга с открытым исходным кодом, предназначенная для отслеживания доступности и производительности ИТ-инфраструктуры — серверов, сетевых устройств, виртуальных машин, приложений и облачных сервисов. Поддерживает сбор метрик через агенты, SNMP, IPMI, JMX и веб-интерфейсы, а также уведомления, визуализацию, автоматизацию и предиктивную аналитику. Разработана Алексеем Владыкиным и поддерживается компанией Zabbix LLC. Пошаговый маршрут — [Практикум Zabbix](/encyclopedia/2-system-network/2-06-sistemnoe-administrirovanie/zabbix-praktikum/intro).

## Zero Downtime Deployment

Развертывание без простоя: метод обновления программного обеспечения, при котором система остаётся полностью доступной для пользователей в процессе замены версий. Достигается с помощью стратегий: blue-green deployment, канареечных релизов (canary releases), rolling updates и использования балансировщиков нагрузки. Критически важен для высоконагруженных и mission-critical приложений с требованиями высокой доступности (SLA ≥ 99.9%).

## ZIP

ZIP: формат архивации и сжатия файлов, разработанный Филом Кацем в 1989 году. Поддерживает сжатие (обычно DEFLATE), шифрование (традиционное и AES), разбиение на тома и хранение метаданных. Стандарт де-факто для передачи групп файлов в ОС Windows, macOS и Linux. Спецификация открыта и поддерживается практически всеми архиваторами.

---

## zlib

zlib: библиотека с открытым исходным кодом для сжатия данных без потерь, реализующая алгоритм DEFLATE. Широко используется в PNG, HTTP (Content-Encoding: deflate/gzip), ZIP и других форматах. Обеспечивает API для сжатия и распаковки в памяти или потоках. Разработана Жан-лу Пейром и Марком Адлером.

---

## ZooKeeper

Apache ZooKeeper: распределённая координационная служба с открытым исходным кодом, предназначенная для управления конфигурациями, синхронизацией, лидер-выбором и распределёнными блокировками в кластерных системах. Обеспечивает согласованное, отказоустойчивое и иерархическое хранилище данных (в виде znode-дерева) с гарантией последовательной согласованности. Используется в Apache Kafka, Hadoop, Solr и других распределённых системах.


---

## Zsh

Z shell (Zsh): расширенная командная оболочка Unix, совместимая с POSIX и bash, но с дополнительными функциями: темизация, автодополнение с меню, глобальные псевдонимы, расширенные шаблоны подстановки, интеграция с Git и мощные возможности настройки. Является оболочкой по умолчанию в macOS начиная с Catalina (10.15).

## ZFS

ZFS (Zettabyte File System) — объединённая файловая система и менеджер томов, разработанный Sun Microsystems (2001–2004).

---

## ZNS (Zoned Namespace)

ZNS (Zoned Namespace) — стандарт NVMe 2.0 для SSD на основе технологии Zoned Storage (зонированное хранилище), в первую очередь SMR-HDD и Z-NAND/Z-SSD.
